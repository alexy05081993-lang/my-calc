# Техническое задание: Интерактивный лид-магнит «Калькулятор потенциала запуска»

Исходя из твоего запроса, делаем веб-приложение для размещения на Replit, Vercel, GitHub Pages или аналогичных платформах. Выбираю **"Калькулятор потенциала запуска"** как наиболее сильный лид-магнит, который бьет в главную боль всех сегментов твоей ЦА.

## **1. Цель (зачем мы это делаем)**

**Бизнес-цель:** Создать интерактивный веб-инструмент, который за 2-3 минуты покажет эксперту или владельцу онлайн-школы конкретную сумму денег, которую они теряют на каждом запуске из-за низких конверсий. Главная задача — конвертировать пользователя, испытавшего шок от суммы упущенной выгоды, в заявку на платный стратегический аудит (25 000 - 35 000 ₽).

**Результат для пользователя:** За 15 минут человек понимает математику своего бизнеса, видит конкретную сумму недополученных денег и получает четкий следующий шаг для решения проблемы.

## **2. Целевая аудитория (для кого)**

**Основные сегменты:**
- Эксперты, наставники, коучи, психологи (28-42 года, 70% женщины)
- Владельцы онлайн-школ с командой (30-48 лет, 50/50)
- База подписчиков от 1000 до 50 000 человек
- Уже имели опыт запусков, но упёрлись в потолок или видят падение результатов

**Ключевые боли:**
- "Не понимаю, сколько реально могу заработать с моими 5000 подписчиков"
- "Ставлю планы запуска с потолка, потом разочаровываюсь"
- "Есть ощущение, что недозарабатываю, но сколько именно — непонятно"
- "Команда работает, отчеты есть, но связь с деньгами размыта"

## **3. Формат и объём (что именно делать)**

**Техническая реализация:**
- Single Page Application (SPA)
- Технологии: HTML5/CSS3/JavaScript (можно Vue.js/React для удобства)
- Адаптивная верстка (Mobile First — 80% трафика из Telegram/Instagram)
- Время загрузки: не более 3 секунд
- Работает без регистрации и ввода email до получения результата

**Структура страницы:** 5 основных блоков, прокручиваемых вертикально

## **4. Ключевой смысл и тон (как мы говорим)**

**Тон коммуникации:**
- Экспертный, прямой, без "воды"
- Партнерский, но уверенный
- Много конкретных цифр, минимум общих слов
- Никаких инфоцыганских обещаний типа "миллионы за 3 дня"

**Ключевое сообщение:** "Математика бизнеса не врет. Если ваши конверсии ниже нормы — вы сжигаете свой бюджет. Вот точная сумма и план решения."

## **5. Ограничения (что нельзя)**

- **Запрещено:** Использовать стоп-слова из гайда "47 стоп-слов" без замены на факты
- **Запрещено:** Требовать email/телефон до показа базового расчета
- **Запрещено:** Сложные анимации, замедляющие загрузку
- **Запрещено:** Размытые обещания ("можете заработать миллионы")
- **Запрещено:** Инфоцыганские формулировки и давление

## **6. Дедлайн**

- **MVP (рабочая математика без финального дизайна):** 5 рабочих дней
- **Финальная версия с дизайном и мобильной адаптацией:** 10 рабочих дней
- **Тестирование на разных устройствах:** +2 дня

## **7. Критерии приёмки (как поймём, что готово)**

Лид-магнит считается готовым, когда:
- Все математические формулы работают корректно при любых введенных значениях
- Страница корректно отображается на iOS и Android
- Дизайн соответствует стилистике гайда "47 стоп-слов"
- Форма заявки успешно отправляет данные
- Нет стоп-слов без замены на конкретику
- Время загрузки не превышает 3 секунд

## **Блок 3: Результаты расчета (динамический)**

**Структура вывода:**

### **Ваши текущие показатели:**
```
База → Заявки: X% (норма: 3-5%)
Заявки → Оплаты: Y% (норма: 8-15%)
Текущая выручка: Z ₽
```

### **Потенциал при нормальных конверсиях:**
```
Потенциальные заявки: [расчет] (база × 4%)
Потенциальные продажи: [расчет] (заявки × 12%)
Потенциальная выручка: [расчет] ₽
```

### **Упущенная выгода (главный триггер):**
```
🚨 ВЫ ТЕРЯЕТЕ [СУММА] ₽ С КАЖДОГО ЗАПУСКА
За год это: [сумма × 4] ₽ недополученной прибыли
```

**Математические формулы для разработчика:**

$$\text{Конверсия в заявку} = \frac{\text{Количество заявок}}{\text{Размер базы}} \times 100\%$$

$$\text{Конверсия в оплату} = \frac{\text{Количество оплат}}{\text{Количество заявок}} \times 100\%$$

$$\text{Потенциальная выручка} = \text{База} \times 0.04 \times 0.12 \times \text{Средний чек}$$

$$\text{Упущенная выгода} = \text{Потенциальная выручка} - \text{Текущая выручка}$$

## **JavaScript логика:**

```javascript
// Основные константы (нормы рынка)
const MARKET_NORMS = {
  leadConversion: 0.04,    // 4% база в заявки
  saleConversion: 0.12,    // 12% заявки в продажи
  minLeadConversion: 0.02, // минимум для диагностики
  minSaleConversion: 0.07  // минимум для диагностики
};

function calculateMetrics() {
  // Получаем данные из формы
  const baseSize = parseInt(document.getElementById('baseSize').value) || 0;
  const leads = parseInt(document.getElementById('leads').value) || 0;
  const sales = parseInt(document.getElementById('sales').value) || 0;
  const averageCheck = parseInt(document.getElementById('averageCheck').value) || 0;
  
  // Валидация данных
  if (baseSize < 100 || averageCheck < 1000) {
    showError("Проверьте введенные данные");
    return;
  }
  
  // Расчет текущих показателей
  const currentLeadConversion = leads / baseSize;
  const currentSaleConversion = sales / leads;
  const currentRevenue = sales * averageCheck;
  
  // Расчет потенциала
  const potentialLeads = Math.round(baseSize * MARKET_NORMS.leadConversion);
  const potentialSales = Math.round(potentialLeads * MARKET_NORMS.saleConversion);
  const potentialRevenue = potentialSales * averageCheck;
  
  // Упущенная выгода
  const lostProfit = Math.max(0, potentialRevenue - currentRevenue);
  
  // Вывод результатов
  updateResults({
    currentLeadConversion: (currentLeadConversion * 100).toFixed(1),
    currentSaleConversion: (currentSaleConversion * 100).toFixed(1),
    currentRevenue: currentRevenue.toLocaleString('ru-RU'),
    potentialRevenue: potentialRevenue.toLocaleString('ru-RU'),
    lostProfit: lostProfit.toLocaleString('ru-RU'),
    yearlyLoss: (lostProfit * 4).toLocaleString('ru-RU')
  });
  
  // Диагностика проблем
  showDiagnosis(currentLeadConversion, currentSaleConversion);
}

function showDiagnosis(leadConv, saleConv) {
  let diagnosis = "";
  
  if (leadConv < MARKET_NORMS.minLeadConversion) {
    diagnosis = "❌ ПРОБЛЕМА В ПРОГРЕВЕ<br>Аудитория не понимает ценность продукта";
  } else if (saleConv < MARKET_NORMS.minSaleConversion) {
    diagnosis = "❌ ПРОБЛЕМА В ПРОДАЖАХ<br>Люди интересуются, но не покупают";
  } else {
    diagnosis = "✅ КОНВЕРСИИ В НОРМЕ<br>Можно масштабировать трафик";
  }
  
  document.getElementById('diagnosis').innerHTML = diagnosis;
}
```
